# Vectum 快速上手

## 1. 项目定位

Vectum 是一个基于 Vector 的轻量级数据管道管理工具，提供三类入口：

- Web UI：基于 Thymeleaf + AMIS 的单页管理界面
- REST API：任务增删改查、启停、日志查看
- MCP Server：把任务管理能力暴露给 AI 客户端调用

项目的核心职责不是复杂业务建模，而是管理多个独立 Vector 进程及其配置、日志和运行状态。

## 2. 技术栈

- Java 17
- Spring Boot 3.2.0
- Spring AI MCP Server 1.1.4
- SpringDoc OpenAPI 2.3.0
- Thymeleaf + AMIS
- Vector 0.35+

## 3. 启动方式

### 本地开发

```bash
mvn clean compile
mvn spring-boot:run
```

默认访问地址：

- Web UI：http://localhost:11002/index
- Swagger：http://localhost:11002/swagger-ui/index.html
- MCP SSE：http://localhost:11002/sse
- Health：http://localhost:11002/actuator/health

### 本地 Vector 前提

开发配置在 `src/main/resources/application-dev.properties`：

```properties
vector.home=vector
task.workspace=workspace
task.file=${task.workspace}/tasks.json
```

因此本地启动任务时，程序会查找：

```text
vector/bin/vector
```

如果只启动 Spring 应用、不真正运行 Vector 任务，可以先不准备 Vector；但点击任务启动会失败。

### Docker

```bash
mvn clean package -DskipTests
docker build -t vectum:latest .
docker run -d -p 11002:11002 vectum:latest
```

生产配置使用：

```properties
vector.home=/vector
task.workspace=/workspace
task.file=${task.workspace}/tasks.json
```

## 5. 核心调用链

### 创建任务

入口：

```text
POST /vectum/api/v1/task/add
```

调用链：

```text
TaskController.add()
  -> TaskServiceImpl.create()
    -> TaskRepository.save()
    -> VectorServiceImpl.createProcess()
```

实际效果：

- 将任务元数据写入 `task.file`
- 创建任务工作目录
- 写入 `push.yaml`、`push.json` 或 `push.toml`
- 写入 Lua 脚本目录（如果请求携带 Lua 文件）

### 启动或停止任务

入口：

```text
POST /vectum/api/v1/task/{id}/toggle
```

调用链：

```text
TaskController.toggle()
  -> TaskServiceImpl.toggle()
    -> VectorServiceImpl.status()
    -> VectorServiceImpl.startProcess() 或 stopProcess()
```

启动时执行的命令形式：

```text
{vector.home}/bin/vector -c {task.workspace}/{taskId}/push.{yaml|json|toml}
```

进程对象缓存在内存中的 `ConcurrentHashMap<String, Process>`。

### 查询任务列表

入口：

```text
GET /vectum/api/v1/task/all
```

调用链：

```text
TaskController.listAll()
  -> TaskServiceImpl.findAll()
    -> TaskRepository.findAll()
    -> VectorServiceImpl.status()
```

返回数据会包含任务元数据和当前运行状态。

### 查看日志

入口：

```text
GET /vectum/api/v1/task/{id}/log?log_type=console
GET /vectum/api/v1/task/{id}/log?log_type=system
```

日志来源：

- `console`：Vector 标准输出，对应 `info.log`
- `system`：Vector 标准错误，对应 `error.log`

日志由 `MonitorServiceImpl.monitorLog()` 异步读取进程输出流并写入文件。

## 6. 数据落盘结构

开发环境默认目录如下：

```text
workspace/
  tasks.json
  {taskId}/
    push.yaml
    pid
    info.log
    error.log
    lua/
```

其中：

- `tasks.json` 保存任务元数据
- `push.*` 保存 Vector 配置
- `pid` 保存最近一次启动的进程 ID
- `info.log` 保存标准输出
- `error.log` 保存标准错误
- `lua/` 保存任务关联的 Lua 脚本

## 8. API 快速参考

基础路径：

```text
/vectum/api/v1/task
```

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/add` | 创建任务 |
| GET | `/all` | 查询所有任务 |
| GET | `/{id}/view` | 查询任务详情 |
| PUT | `/{id}` | 更新任务 |
| DELETE | `/{id}` | 删除任务 |
| DELETE | `/batch?ids=1,2` | 批量删除 |
| PUT | `/batch?ids=1,2` | 批量更新 |
| POST | `/{id}/toggle` | 启动或停止任务 |
| GET | `/{id}/log?log_type=console` | 查看控制台日志 |
| GET | `/{id}/log?log_type=system` | 查看系统日志 |

统一响应格式：

```json
{
  "status": 0,
  "msg": "请求成功",
  "data": {}
}
```

日志接口直接返回文本内容。

## 9. MCP 能力

MCP Server 配置在 `application-dev.properties` 和 `application-prod.properties`：

```properties
spring.ai.mcp.server.sse-endpoint=/sse
spring.ai.mcp.server.sse-message-endpoint=/mcp/message
spring.ai.mcp.server.capabilities.tool=true
```

工具方法定义在 `McpTaskTools`，主要包括：

- `getTasks`
- `getTask`
- `createTask`
- `updateTask`
- `deleteTask`
- `toggleTask`
- `getTaskLog`
- `batchDeleteTasks`

这些工具最终复用 `TaskService`，和 REST API 操作的是同一套任务数据。

## 10. 常见修改点

### 新增任务字段

通常需要同步修改：

- `Task`
- `TaskDto`
- `TaskVo`
- `Task.updateFromDto()`
- `templates/index.html`

如果字段需要 MCP 支持，还要修改 `McpTaskTools`。

### 新增 API

通常修改：

- `TaskController`
- `TaskService`
- `TaskServiceImpl`

如需持久化新行为，再修改 `TaskRepository` 或 `TaskRepositoryImpl`。

### 修改 Vector 启动逻辑

重点看：

- `VectorServiceImpl.startProcess()`
- `VectorServiceImpl.stopProcess()`
- `VectorServiceImpl.init()`

这些方法负责进程启动、停止和应用重启后的任务恢复。

### 修改前端页面

页面是 AMIS JSON 配置，主要在：

```text
src/main/resources/templates/index.html
```

当前页面包含：

- 创建任务 Dialog
- 任务列表 CRUD
- 启动/停止按钮
- 编辑 Dialog
- 日志查看 Dialog
- 删除按钮

## 11. 当前需要注意的实现细节

1. 任务元数据使用 JSON 文件存储，不是数据库。
2. 运行态进程存在内存 Map 中，服务重启后依赖工作目录和 pid 文件恢复。
3. `VectorServiceImpl.init()` 会扫描工作目录，发现 pid 文件后尝试 kill 旧进程再重新启动。
4. `TaskRepositoryImpl` 内部直接创建 `ObjectMapper`，不使用全局 Jackson 配置。
5. REST 层会抛出 `ApiException`，但当前项目没有全局 `@ControllerAdvice`。
6. `McpTaskTools.updateTask()` 看起来支持部分字段更新，但底层 `Task.updateFromDto()` 会整体覆盖字段，修改时需要特别小心。
7. 本地 `mvn test` 可能因为首次下载 Maven 依赖或网络问题失败；源码编译可先用 `mvn clean compile` 验证。

## 12. 推荐阅读顺序

新接手时建议按这个顺序读代码：

1. `TaskController`
2. `TaskServiceImpl`
3. `VectorServiceImpl`
4. `TaskRepositoryImpl`
5. `MonitorServiceImpl`
6. `McpTaskTools`
7. `templates/index.html`

读完这几处，就能理解任务从页面/API 创建，到配置落盘、进程启动、日志展示、MCP 调用的完整流程。
